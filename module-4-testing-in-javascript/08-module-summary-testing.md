You should now be able to:

- **Differentiate** between manual and automated testing

  1. **Unit Testing** - Testing smallest units of code in isolation
  2. **Integration Testing** - Testing how parts work together
  3. **End-to-End (E2E) Testing** - Testing complete user workflows

You explored features available to web developers within Jest:

- **Code coverage** - Measuring what percentage of code is tested

- **Snapshot testing** - Verifying no regressions in the DOM after code changes
- **Asynchronous testing** - Testing async code with minimal setup

## The Testing Pyramid (Recap)

```
         /\
        /E2E\         ← Fewest tests, slowest, most expensive
       /------\
      /  Integ \      ← Moderate tests, medium speed & cost
     /----------\
    /   Unit     \    ← Most tests, fastest, least expensive
   /______________\
```

## Key Takeaways

### Why Testing Matters

- **Prevents bugs** from reaching production
- **Saves time** by catching issues early
- **Provides confidence** when making changes
- **Serves as documentation** for how code should behave
- **Enables refactoring** safely

### Testing Best Practices

1. Write tests for all new code
2. Follow the testing pyramid structure
3. Use TDD when appropriate
4. Aim for high code coverage
5. Keep tests simple and focused
6. Run tests frequently
7. Use mocking to isolate dependencies
